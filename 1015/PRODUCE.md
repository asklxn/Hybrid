### 📱 협업 도구 및 방법

#### 1. **Discord** (주요 협업 플랫폼)

**사용 목적**:
- ✓ 실시간 화면 공유 및 코드 리뷰
- ✓ 파일 공유 (HTML, CSS, JS, 이미지)
- ✓ 음성 채팅 (야간 작업 세션)
- ✓ 코드 스니펫 공유

**채널 구성**:
```
#general     - 일반 대화
#dev         - 개발 관련
#design      - 디자인 피드백
#bugs        - 버그 리포트
```

**선택 이유**: 
> "저희 셋 다 학교랑 개인일정 때문에 바빠서 직접 만나기 어려웠습니다. 그래서 Discord로 실시간 소통하면서 작업했고, 밤늦게까지 음성 채팅하면서 코딩하고 화면 공유로 바로바로 피드백 주고받았습니다."

#### 2. **KakaoTalk** (빠른 소통)

**사용 목적**:
- ✓ 긴급 알림 및 빠른 의사결정
- ✓ 모바일 스크린샷 공유
- ✓ 회의 일정 조율
- ✓ 간단한 버그 제보

**선택 이유**:
> "급하게 확인해야 할 일이나 간단한 수정 사항은 카톡이 제일 빨랐습니다. '이거 색 좀 바꿔봐' 이런 식으로 바로바로 소통했습니다."

#### 3. **GitHub** (버전 관리)

**사용 목적**:
- ✓ 코드 버전 관리
- ✓ GitHub Pages 배포
- ✓ 작업 이력 추적
- ✓ 코드 백업

### 💬 회의 내용

#### 📅 Day 1: 프로젝트 킥오프 (2024.11.08)
**시간**: 저녁 8시 ~ 11시 (3시간)  
**장소**: Discord 음성 채팅  
**참석자**: 임선우, 권법진, 김종현

**논의 내용**:

1. **프로젝트 목표 설정**
   ```
   김종현: "야, 우리 박물관 사이트 어떻게 만들래?"
   
   권법진: "오 좋지. 근데 1주일 안에 가능해?"
   
   임선우: "AI 쓰면 될 듯. Claude랑 ChatGPT 활용하면 
           금방 만들 수 있을 거 같은데?"
   
   권법진: "ㅇㅇ 그럼 API 같은 거 말고 순수하게 
           HTML, CSS, JS만으로 해보자."
   ```

2. **역할 분담**
   ```
   김종현: "내가 기획이랑 회의록은 다 쓸게. 
           GitHub 관리도 내가 할게."
   
   권법진: "그럼 나는 메인 페이지랑 CSS, JS 전체를 담당할게."
   
   임선우: "ㅇㅋ. 그럼 나는 전체적으로 봐주면서 서브느낌으로 할게
            Github랑 문서화작업까지"
   
   권법진: "굿. 코드 퀄리티는 내가 책임질게."
   ```

3. **기술 스택 결정**
   ```
   권법진: "Android Studio로 하면 될까?"
   
   김종현: "아니 VSCode로 하자 간편하잖아. 
            어차피 API도 안 쓰기로 했잖아."
   
   권법진: "ㅇㅋ 그럼 Vanilla JS로 가자. 
           대신 ES6+ 문법은 써도 되지?"
   
   임선우: "당연하지. 그리고 반응형은 필수야."
   ```

4. **개발 환경**
   ```
   임선우: "VS Code랑 Claude.ai 쓰지?"
   
   권법진: "나 Cursor 써봤는데 진짜 좋더라. AI 자동완성 개쩔어."
   
   김종현: "오 그거 써보자. 그리고 나중에 Android Studio로 
            앱으로도 만들어볼까?"
   
   권법진: "Cordova로 하면 될 듯. 일단 웹부터 완성하자."
   ```

**결정 사항**:
- ✅ API 없이 순수 HTML/CSS/JS 사용
- ✅ AI 도구 적극 활용 (Claude, ChatGPT, Cursor)
- ✅ 1주일 내 완성 목표
- ✅ 중학생도 따라할 수 있게 문서화

#### 📅 Day 2: 디자인 및 메인 개발 (2024.11.09)
**시간**: 저녁 11시 ~ 새벽 2시 (3시간)  
**장소**: Discord 음성 채팅 + 화면 공유  
**참석자**: 임선우, 권법진, 김종현

**논의 내용**:

1. **색상 팔레트 선택**
   ```
   권법진: "금색 어때? 박물관 느낌 나잖아."
   
   김종현: "오 좋은데 너무 진하면 촌스러울 수 있어."
   
   권법진: "#FFD700 이건 어때?" (화면 공유)
   
   임선우: "음... 좀 밝은데? #D4AF37 이게 더 고급스러운 듯."
   
   권법진: "ㅇㅇ 이게 낫네. 이거로 가자."
   ```

2. **히어로 섹션 논의**
   ```
   권법진: "메인에 뭐 넣을까? 이미지?"
   
   김종현: "YouTube 영상 어때? 박물관 홍보 영상 넣으면 
            역동적일 것 같은데."
   
   권법진: "오 좋네. 자동재생 시키고."
   
   임선우: "근데 모바일에서는 영상이 너무 크지 않을까?"
   
   권법진: "그럼 모바일에서는 숨기자. 미디어 쿼리로."
   ```

3. **AI 프롬프트 작성**
   ```
   김종현: "Claude한테 어떻게 물어볼까?"
   
   권법진: "최대한 구체적으로 물어봐야 해. 
           '박물관 웹사이트 만들어줘' 이러면 
           원하는 대로 안 나와."
   
   임선우: "그럼 '국립중앙박물관 히어로 섹션을 만들어줘. 
           왼쪽에는 제목, 오른쪽에는 YouTube 영상...' 
           이런 식으로?"
   
   권법진: "ㅇㅇ 정확해. 색상 코드, 크기, 
           모든 걸 다 알려줘야 해."
   ```

**결정 사항**:
- ✅ 금색(#D4AF37) + 빨간색(#C60C30) 조합
- ✅ 히어로 섹션에 YouTube 영상
- ✅ 그리드 + 플렉스 혼합 레이아웃
- ✅ AI 프롬프트는 최대한 구체적으로

**결정 사항**:

### 🎨 의사결정 과정

#### 🔄 주요 수정 사항 (3~5회 반복)

##### 1. 색상 변경 (4회)
```
1차: #FFD700 (순금색)
→ "너무 밝아서 눈이 부셔요." (임선우)

2차: #C4A000 (어두운 금색)
→ "이건 너무 어둡고 촌스러워요." (권법진)

3차: #DAA520 (골든로드)
→ "아직도 좀 밝은데..." (김종현)

4차: #D4AF37 (메탈릭 골드) ✅
→ "완벽해요!" (전원 동의)
```

##### 2. 히어로 섹션 레이아웃 (3회)
```
1차: 중앙 정렬, 텍스트만
→ "너무 심심한데? 뭔가 더 필요할 것 같아." (권법진)

2차: 배경 이미지 + 텍스트
→ "이미지가 텍스트 가독성을 떨어뜨려." (김종현)

3차: 좌우 분할 (텍스트 + 영상) ✅
→ "역동적이고 좋아요!" (전원 동의)
```

##### 3. 모바일 메뉴 (3회)
```
1차: 사이드바 슬라이드 메뉴
→ "화면을 너무 많이 가려서 답답해." (임선우)

2차: 풀스크린 오버레이 메뉴
→ "너무 과하지 않아? 간단한 게 나을 듯." (권법진)

3차: 햄버거 드롭다운 메뉴 ✅
→ "깔끔하고 직관적이에요!" (전원 동의)
```
